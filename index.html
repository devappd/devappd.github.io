Roly Poly Putt

2020
C++ HTML5 JS WebAssembly (Emscripten)

Roly Poly Putt is a wacky physics-based 3D platformer released for iOS and Android where YOU are the golf ball! As a proof-of-concept, I ported the C++ game engine to run in web browsers via WebAssembly. I addressed the technical gap between how a traditional game is structured and how browsers operate. I designed a custom CMake-based build pipeline to automate compilation and asset packaging, bundling game resources into a browser-accessible virtual filesystem.

Beyond the build system, I made targeted adaptations to the engine's C++ source. I replaced synchronous networking calls with async equivalents to keep the UI responsive and rerouted filesystem operations through browser-compatible APIs since standard directory scanning doesn't function in the sandboxed web environment. I resolved audio compatibility challenges by offloading decoding to the browser's native engine, reducing WebAssembly runtime overhead.

I released a few of these improvements on my GitHub profile.

Sonic Robo Blast 2

2018 - 2020
C HTML5 JS WebAssembly (Emscripten)

Sonic Robo Blast 2 is a 3D Sonic the Hedgehog fangame with over two decades of community history. Originally built in C as a desktop-only application, I ported it to run in the web browser by using the Emscripten compiler. I adapted the game's asset loading and audio subsystems to operate within the browser's constraints. This included loading textures in an on-demand fashion and restructuring the audio system to support streaming music delivery and stable playback across environments.

The result has run on PC, Chromebook, iPhone, and Android without installation. To make that practical, I built an asset streaming pipeline that gets players into the game before the full download completes, a local storage layer that eliminates re-downloads on repeat visits, and an installable app experience for mobile devices. The interface adapts to the player's platform -- adjusting controls, instructions, and available quality settings -- with a separate low-end build targeting devices where memory and performance are constrained.

Besides these features, I also contributed as a gameplay programmer.
